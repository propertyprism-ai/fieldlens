# fieldlens — spec.md

## 产品定位
给 Hard Money Lender 提供房产风险情报报告。
输入一个 Zillow URL，输出一份可直接用于 pitch 的风险调查报告。

---

## 核心流程

```
输入: Zillow URL
    ↓
[Step 1] Apify 拉取房产数据
    Actor: maxcopell/zillow-detail-scraper
    输出: 结构化 JSON（地址/价格/税/租金/描述/价格历史等）

    ↓

[Step 2] Layer 1 — 宏观区域扫描
    Reflexion agentic_call()
    输入: 地址 → City + Zip + 街区名
    调查范围:
      - 近期犯罪趋势（街区级，非城市统计）
      - 市场走向（价格趋势、空置率、人口流动）
      - 监管风险（租管条例、待颁政策）
      - 基础设施风险（洪水区、污染、噪音）
      - 宏观经济信号（就业、人口）
    输出: 区域风险 JSON + 风险评级（LOW/MODERATE/HIGH/CRITICAL）

    ↓

[Step 3] Layer 2 — 微观房产法证
    Reflexion agentic_call()
    输入: Zillow JSON + Layer 1 区域背景（注入 prompt）
    调查范围（6-Phase）:
      - Phase 1: 业主溯源（Flipper/LLC/REO/税务拍卖）
      - Phase 2: Permit 审计（装修许可 vs 实际施工）
      - Phase 3: 犯罪安全（街区级具体事件）
      - Phase 4: 基础设施（洪水/污染/噪音/下水道）
      - Phase 5: 租金真实性（Zestimate vs 市场 vs HUD FMR）
      - Phase 6: 市场监管风险（code violation/HOA/税务重估）
    输出: 房产风险 JSON + KILL/PURSUE 判决

    ↓

[Step 4] 报告生成
    合并 Layer 1 + Layer 2 → Markdown 报告
    报告结构:
      - 封面（地址/日期/综合风险评级）
      - 区域概况（宏观，3-5个关键发现）
      - 房产法证（微观，6-Phase 详情）
      - 综合判决（KILL/PURSUE + Top 3 风险 + Top 2 亮点）
      - 免责声明
    输出: results/{zpid}_{date}.md
```

---

## 验收标准（MVP）

1. `python main.py --url <zillow_url>` 能跑通全流程
2. Apify 成功返回房产结构化数据
3. Layer 1 输出包含：区域风险评级 + 至少 2 个具体发现（含来源引用）
4. Layer 2 输出包含：6-Phase 调查结果 + KILL/PURSUE 判决
5. 生成 Markdown 报告文件，包含上述全部内容
